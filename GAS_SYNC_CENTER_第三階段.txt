1. 新版 UI / Firebase 主線維持不破壞。

3. 先建立同步中心骨架與規則，不直接在前端硬做危險寫入。

既有 Firebase 主線集合：
- products

- staff
- orders
- order_items
- inventory
- inventory_logs
- payments
- shipping
- refunds
- sales_report
- sync_meta

重要同步原則：
- 集合不存在時，Firestore 會在第一次寫入文件時自動建立。
- 文件存在則更新，不存在則新增。
- inventory = 結果；inventory_logs = 事實來源。
- products 文件 ID = 商品編號。
- inventory 文件 ID = 商品編號。
- orders 文件 ID = 訂單編號。
- order_items 建議 ID = 訂單編號_商品編號_序號。
- inventory_logs 需以 eventId 防重。
